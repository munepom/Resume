(前職、株式会社Aにて)
自社製品である現場管理アプリケーションの Android 版アプリ開発に途中からジョインし、既存不具合の修正と新規機能の開発を行いました。

苦労した点
1. 既存コードのバグ修正
1 名体制で開発していたところに私が加わり、 2 名体制での開発となりました。
しかし、既存コードがいわゆるオレオレ Thread や Queue を駆使していたもので、データの一時保存も独自の仕組みを利用していて揮発してしまうというバグを抱えたものでした。

そこで、 Android の標準的な仕組みを理解して実装しましょうという方針にして、
アプリのライフサイクルに合わせた適切な Thread の利用、
一時保存データは Parcelable を利用しておくよう改良し、バグを解消しました。

今改良を行うなら、 RxJava や Realm などの現代的なフレームワークを導入し、データの流れに応じて View が更新されるという設計にすることで解決すると思われます。

2. カメラ機能の改良
Android 標準カメラをアプリから利用すると、 Android の Offical API から取得できるはずの画像情報が端末メーカーや OS によって変動してしまうという問題がありました。
（古い Android 端末だと、画像自体を取得できない場合があるといった挙動に悩まされました。）
端末ごとの挙動を確認し、極力 Offical API から画像情報を取得できるようにはなりました。
根本解決のため、別開発者が独自カメラの開発を行いました。

3. Android 4.x への対応
Material Design への対応、 Activity から Fragment への置き換えを行いました。
Android 2.2 以降をサポート対象としていたため、 Support Library を使い、可能な範囲で対応しました。
