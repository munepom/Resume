前職である株式会社Aにて、自社製品の Web アプリケーションのメンテナンスやカスタマイズを行いました。

エンジニアが数名しか在籍していなかったので、
フロントエンドもサーバサイドも開発し、
週に一度はサーバーメンテナンスを行い、
リリースも実施するマルチロールエンジニア的な役割でした。

フロントエンドは JSP と jQuery を利用して開発を行いました。
サーバサイドは Java 製の独自社内フレームワークで開発を行いました。

主な実績
1. JavaScript 製のスケジュール管理画面を作成しました。
jQuery を利用し、カレンダー日付を行、工務店を列とした Excel ライクな画面をイメージしました。

2. 現場写真閲覧画面の SPA 化を行いました。
jQuery, jQuery UI を利用し、検査保険業務向けの大規模カスタマイズで実現できました。
JSP をテンプレート化せずというのが会社のやり方でしたが、極力テンプレート化し、サーバサイドレンダリングを行いやすくする工夫をしました。
（今作り直すなら、 Vue.js や React で JavaScript コンポーネント化を目指していると思います。。。）

3. Web カメラからの画像登録対応
現場に設置された Web カメラからの画像登録に対応するため、 FTP 経由でサーバに送信された画像データを処理できる仕組みを構築した。
メール経由で送信された画像は Java 製の自社 worker で対応できますが、
仕組みが存在しなかったので inotify でディレクトリ監視を行い登録できるようにしました。

4. Gradle の導入
Makefile による javac 逐次呼び出し方式を Gradle へ置き換えました。
30 分は要したビルド時間を 30 秒程度まで短縮できました。

5. Redmine の導入
プロジェクトの状態把握が紙ベースでやりづらかったので、
Redmine を導入し、進捗を技術メンバ全員と共有できるようにしました。
Subversion の履歴と紐付けることで、バグも追跡しやすくしました。
